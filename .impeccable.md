## Design Context

### Users
Primary users are ham radio operators running a dedicated transmitter appliance based on a Raspberry Pi and companion hardware. They are expected to be technically capable, comfortable with experimental transmission modes, and willing to work with precise controls rather than heavily simplified abstractions.

The main jobs in this UI are configuring and operating transmit-only workflows for WSPR, QRSS, FSKCW, and DFCW modes, along with reviewing logs, maintenance state, and spot data.

### Brand Personality
Precise, clean, technical.

The interface should feel trustworthy and utilitarian rather than decorative. It should support middle-aged technical users who use computers pragmatically and want low-friction control surfaces, clear state, and direct language.

### Aesthetic Direction
Technical and restrained, not flashy. The UI should preserve support for both light and dark themes and keep the visual language clean, readable, and operationally focused. It should avoid ornamental trends and instead present an instrument-panel feel with strong information hierarchy and obvious system state.

### Design Principles
1. Prioritize operational clarity over visual novelty.
2. Make system state, mode, and transmission context easy to scan quickly.
3. Use visual hierarchy to support technical decision-making, not decoration.
4. Preserve a clean light/dark experience without changing interaction models.
5. Keep controls and terminology direct, consistent, and pragmatic.
